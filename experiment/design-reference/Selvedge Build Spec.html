01 · Product overview

What we’re building

ProblemHigh-end fashion factories landfill billions of tons of pristine leftover fabric (deadstock) — runs too small or irregular for large brands to use.
SolutionA marketplace where garment factories list premium textile leftovers for sale to indie designers, students and boutique upcycling brands.
Unique edgeFrictionless listing: a worker photographs a roll; AI estimates material, texture, colour family and approximate yardage and generates the listing instantly.
Why nowEU EPR mandates (2025), anti-destruction laws and Digital Product Passports (2027) turn textile waste from a write-off into a fine. Mills need a circular outlet with a paper trail.
UsersBuyers: sustainable fashion brands, design students, upcyclers. Sellers: textile factories & mills.
Revenue10% processing fee on every secondary fabric sale. €0 to list. Factory receives 90%, paid weekly.

Voice & copy rules

  • Confident, declarative, a little editorial. Headlines in ALL CAPS with one word swapped to Bodoni italic for contrast: “RESCUE THE cloth NOBODY COULD USE.”
  • Numbers carry the argument (92M tonnes, 90% payout, 48H to live listing). Use sparingly, big.
  • Never say “eco-friendly” or “green”. Say rescued, salvaged, second cut, deadstock.
  • Microcopy in Space Mono, uppercase, letterspaced — reads like a spec label.

Fabric colour data (listing chips)

18 named fabric colours, each with a colour family used by the browse filter: Neutral, Grey, Green, Blue, Warm, Red, Pink, Purple. Each has base/hi/lo values driving the procedural weave render — full table in the data file (src/data.jsx in the prototype project).

  • Badge is decorative — no link.
  • All CTAs route: Browse → #/browse, List inventory → #/upload.

06 · Page blueprint — the magic moment

Factory upload #/upload

A 4-state machine on a field-dark page with a centered bone “stage” card. Stepper: Photograph → AI drafts it → Publish.

StateContents & behaviour
chooseDashed drop-zone (“Photograph a fabric roll” · camera icon · opens file picker; any image accepted) + 6 sample-roll tiles for the demo path. Below the stage: 3 feature stats (~8 seconds · 4 specs auto · 90% to you).
analyzingTwo-pane: left = photo with scan-line sweep, corner reticles, grid overlay; right = progress bar 0→100% (~4.4s) with 5 phase lines that check off in sequence: Reading weave structure → Classifying fibre & material → Sampling colour family → Estimating roll yardage → Drafting your listing.
reviewLeft = photo + confidence card (e.g. 94%, animated bar, Retake). Right = form with every field pre-filled and tagged with an “AI” chip: title, material (select), texture, colour swatch-picker (18 colours), yardage/width/weight numeric, suggested price. Live payout calc: You receive (90%) = price × yards × 0.9. Publish button (clay).
publishedCheck pop animation · “You’re live.” · the resulting listing card preview + “Live now” tag, listed time, potential payout · CTAs: List another roll / See it in the marketplace.

Rules

  • In the prototype the “AI” result comes from the chosen sample’s data after a staged delay; a real build calls a vision API and maps its response onto the same field set.
  • Worker never types first — every field arrives pre-filled; typing is correction, not entry. That’s the core product promise.
  • Confidence ≥90% renders clay; you may flag <88% fields for review tint in production.
  • Publishing should append to the marketplace dataset so the new listing appears in #/browse.
Demo timing: progress ticks every 95ms (+2–6%), phases advance at 20% intervals, review reveals at ~4.4s. Keep it under 5s — the speed is the pitch.

Derived & persisted

  • Colour family comes from the colour token table (Neutral/Grey/Green/Blue/Warm/Red/Pink/Purple) — the browse filter facets on it.
  • Favourites: localStorage["selvedge_favs"], JSON array of ids.
  • Payout = price × yards × 0.9 (10% platform fee). Shown in upload review + published states.
  • Sort keys: addedDays asc (newest), price asc/desc, yards desc.

Build checklist

  • Hash router, 4 routes; scroll-to-top on change; nav state highlights.
  • Shared components: Nav, Footer, ListingCard, FabricSwatch (texture), Icon set (thin line, 1.6 stroke).
  • Responsive: 1180 / 940 / 860 / 620 / 560 breakpoints (grid steps, filter drawer, nav burger).
  • Reduced motion: kill marquee + reveals under prefers-reduced-motion.
  • Accessibility: 44px+ hit targets, focus-visible rings (2px clay), alt text on fabric images, labels on all filter controls.
  • Min text sizes: 12px UI floor; body 16px+.

Reference implementation

A working HTML/React prototype of all four pages (Campaign direction: Selvedge — Directions.html, full site: Selvedge.html) exists in the design project — use it as the source of truth for spacing, copy and the procedural weave CSS. Lift the weave engine (fabricBg()) and full 18-listing dataset (src/data.jsx) verbatim.

North star: it should feel like a printed campaign poster that happens to be shoppable — hard colour cuts, huge caps, one italic wink per headline, mono spec labels everywhere.
